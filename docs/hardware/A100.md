# NVIDIA A100

## Role in this repo

A100 is the baseline modern data-center GPU target for CUDA kernels in this
library. It is useful for validating Ampere-era behavior before optimizing for
Hopper or Blackwell systems.

## CUDA target

| Field | Value |
| --- | --- |
| Architecture | NVIDIA Ampere |
| Compute capability | 8.0 |
| `nvcc` arch | `sm_80` |
| Common forms | PCIe, SXM |
| Primary workload fit | CUDA fundamentals, FP32/FP64 HPC, TF32/FP16 tensor workloads, MIG partitioning tests |

## Kernel-development notes

- Treat A100 as the oldest data-center target in this hardware set.
- Prefer `-arch=sm_80` when building A100-specific cubins.
- Use A100 to test kernels that should still run without Hopper-only or
  Blackwell-only instructions.
- Memory bandwidth is high enough that simple bandwidth kernels should report
  roofline-style effective GB/s, not only elapsed time.
- Tensor workloads should separate CUDA core, Tensor Core, TF32, FP16/BF16,
  and sparse paths where possible.
- MIG can change visible memory size and scheduling behavior, so benchmark
  output should record `nvidia-smi` state and device properties.

## Benchmark priorities

- SAXPY/vector bandwidth and copy bandwidth.
- Reductions and scans using shared memory and warp reductions.
- GEMM and batched GEMM with TF32 and FP16/BF16.
- Occupancy experiments around registers, shared memory, and block size.
- PCIe versus NVLink transfer tests when multiple A100s are available.
