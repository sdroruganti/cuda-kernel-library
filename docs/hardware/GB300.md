# NVIDIA GB300

## Role in this repo

GB300 is the Blackwell Ultra target. It is relevant for the most advanced AI
reasoning, long-context inference, and FP4-heavy kernels in this repository.
Compared with GB200, it raises the importance of attention performance and HBM
capacity.

## CUDA target

| Field | Value |
| --- | --- |
| Architecture | NVIDIA Blackwell Ultra |
| Compute capability | 10.3 |
| `nvcc` arch | `sm_103` or `sm_103a` when using Blackwell Ultra architecture-specific features |
| Common forms | GB300 NVL72, GB300 DGX Station |
| Primary workload fit | Reasoning inference, long-context attention, FP4/FP8 tensor kernels, rack-scale training and inference |

## Useful specs

| Spec | GB300 NVL72 |
| --- | ---: |
| Configuration | 72 Blackwell Ultra GPUs, 36 Grace CPUs |
| Fast memory | 37 TB |
| GPU memory and bandwidth | 20 TB HBM3E, up to 576 TB/s |
| CPU memory and bandwidth | 17 TB LPDDR5X, 14 TB/s |
| CPU cores | 2,592 Arm Neoverse V2 cores |
| NVLink bandwidth | 130 TB/s |
| FP4 Tensor Core | 1,440 PFLOPS sparse, 1,080 PFLOPS dense |
| FP8/FP6 Tensor Core | 720 PFLOPS sparse |
| INT8 Tensor Core | 24 POPS sparse |
| FP16/BF16 Tensor Core | 360 PFLOPS sparse |
| TF32 Tensor Core | 180 PFLOPS sparse |
| FP32 | 6 PFLOPS |
| FP64 / FP64 Tensor Core | 100 TFLOPS |

## Kernel-development notes

- Use GB300 for kernels that need Blackwell Ultra behavior or that are designed
  around long-context reasoning workloads.
- Prefer `sm_103` for broad GB300 builds. Use `sm_103a` only when relying on
  architecture-specific Blackwell Ultra instructions.
- Attention, KV-cache, and decode kernels should report context length,
  generated-token count, batch size, and precision.
- Compare GB300 results against GB200 only when the benchmark is normalized for
  GPU count, memory capacity, and rack topology.
- GB300 systems are liquid-cooled rack systems or high-end workstation systems,
  so benchmark logs should include power limit, thermals, and clock state.

## Benchmark priorities

- Long-context attention and KV-cache read/write bandwidth.
- FP4, FP6, FP8, FP16/BF16, and TF32 GEMM kernels.
- Fused inference kernels: RMS norm, rotary embedding, activation, softmax,
  quantization/dequantization, and sampling helpers.
- Multi-GPU collectives across the 72-GPU NVLink domain.
- Throughput per watt and latency per generated token.
