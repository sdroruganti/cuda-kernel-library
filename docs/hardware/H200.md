# NVIDIA H200

## Role in this repo

H200 is the Hopper-generation high-memory target. For this repository it is the
main comparison point for memory-bound kernels, FP8/Tensor Core work, and
large-problem benchmarks that outgrow A100.

## CUDA target

| Field | Value |
| --- | --- |
| Architecture | NVIDIA Hopper |
| Compute capability | 9.0 |
| `nvcc` arch | `sm_90` or `sm_90a` when using Hopper architecture-specific features |
| Common forms | SXM, NVL PCIe |
| Primary workload fit | LLM inference/training kernels, memory-bandwidth tests, FP8 tensor workloads, HPC |

## Useful specs

| Spec | H200 SXM | H200 NVL |
| --- | ---: | ---: |
| GPU memory | 141 GB | 141 GB |
| Memory bandwidth | 4.8 TB/s | 4.8 TB/s |
| FP64 | 34 TFLOPS | 30 TFLOPS |
| FP64 Tensor Core | 67 TFLOPS | 60 TFLOPS |
| FP32 | 67 TFLOPS | 60 TFLOPS |
| TF32 Tensor Core | 989 TFLOPS sparse | 835 TFLOPS sparse |
| FP16/BF16 Tensor Core | 1,979 TFLOPS sparse | 1,671 TFLOPS sparse |
| FP8 Tensor Core | 3,958 TFLOPS sparse | 3,341 TFLOPS sparse |
| TDP | Up to 700 W configurable | Up to 600 W configurable |
| MIG | Up to 7 instances, about 18 GB each | Up to 7 instances, about 16.5 GB each |
| Interconnect | NVLink 900 GB/s, PCIe Gen5 | NVLink bridge up to 900 GB/s per GPU, PCIe Gen5 |

## Kernel-development notes

- Prefer H200 for testing kernels that are limited by memory capacity or memory
  bandwidth.
- Use `sm_90` for portable Hopper cubins. Use `sm_90a` only when the kernel
  intentionally depends on architecture-specific Hopper instructions.
- FP8 paths should be benchmarked separately from FP16/BF16 paths.
- Shared-memory, register-pressure, and occupancy experiments should be logged
  with the exact GPU form factor because SXM and NVL systems have different
  power and topology constraints.
- Large-context or large-batch kernels should report working-set size because
  H200's 141 GB memory materially changes the feasible problem sizes.

## Benchmark priorities

- HBM bandwidth kernels across contiguous, strided, and irregular access.
- FP8, FP16, BF16, TF32, and FP64 GEMM microbenchmarks.
- Attention primitives: softmax, layer norm/RMS norm, reductions, KV-cache
  reads, and fused memory-bound kernels.
- Multi-GPU collectives and peer-copy tests on NVLink-capable systems.
- Power-capped runs when comparing performance per watt.
